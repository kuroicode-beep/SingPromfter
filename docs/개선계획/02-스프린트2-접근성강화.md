## 3. 상세 작업 지시

### 🔴 S2-C1: 슬라이더 +/- 버튼 보조 조작 추가

- **우선순위**: P0 (저시력 사용자 직격)
- **대상 파일**: `lib/screens/song_list_screen.dart`, `lib/screens/prompter_screen.dart`
- **대상 위치**:
  - `_MiniSlider` 위젯 (`song_list_screen.dart:1879-1922`)
  - `_InlineSlider` 위젯 (`prompter_screen.dart:340-390`)
  - 볼륨 슬라이더 (`song_list_screen.dart:1400-1421`)

#### 문제
글자 크기/줄간격/속도/볼륨 모두 가는 슬라이더(`trackHeight: 3`, `thumbRadius: 6`). 시력이 나쁜 사용자가 드래그하여 조작하기 어려움.

#### 작업 지시
1. 각 슬라이더 양옆에 `-` / `+` 버튼(최소 48×48dp)을 추가한다.
2. 버튼은 한 단계씩 증감(`divisions` 단위).
3. `_MiniSlider`, `_InlineSlider` 위젯에 선택적 `onDecrement`/`onIncrement` 콜백 추가.
4. 볼륨 슬라이더에도 동일 패턴 적용.

#### 구현 예시
```dart
class _MiniSlider extends StatelessWidget {
  final String label;
  final double value;
  final double min;
  final double max;
  final int? divisions;
  final ValueChanged<double> onChanged;
  final double step; // 단계 크기

  const _MiniSlider({
    required this.label,
    required this.value,
    required this.min,
    required this.max,
    required this.onChanged,
    this.divisions,
    this.step = 1.0,
  });

  double get _step => divisions != null ? (max - min) / divisions! : step;

  void _decrement() {
    final next = (value - _step).clamp(min, max);
    onChanged(next);
  }

  void _increment() {
    final next = (value + _step).clamp(min, max);
    onChanged(next);
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisSize: MainAxisSize.min,
      children: [
        Text(label, style: const TextStyle(color: AppColors.textMuted, fontSize: 10)),
        Row(
          children: [
            _StepButton(icon: Icons.remove, onTap: _decrement),
            Expanded(
              child: SliderTheme(
                data: SliderThemeData(
                  trackHeight: 4, // 3 → 4로 약간 두껍게
                  thumbShape: const RoundSliderThumbShape(enabledThumbRadius: 10), // 6 → 10
                  overlayShape: const RoundSliderOverlayShape(overlayRadius: 16),
                ),
                child: Slider(
                  min: min, max: max, divisions: divisions,
                  value: value, onChanged: onChanged,
                ),
              ),
            ),
            _StepButton(icon: Icons.add, onTap: _increment),
          ],
        ),
      ],
    );
  }
}

class _StepButton extends StatelessWidget {
  final IconData icon;
  final VoidCallback onTap;
  const _StepButton({required this.icon, required this.onTap});

  @override
  Widget build(BuildContext context) {
    return SizedBox(
      width: 44,
      height: 44,
      child: IconButton(
        icon: Icon(icon, size: 20, color: AppColors.textPrimary),
        onPressed: onTap,
        tooltip: '조정',
      ),
    );
  }
}
```

#### 완료 기준 (DoD)
- [ ] 모든 슬라이더(크기/줄간격/속도/볼륨)에 +/- 버튼이 있다
- [ ] 버튼 탭 시 한 단계씩 증감한다
- [ ] 슬라이더 trackHeight가 4 이상, thumb radius 10 이상으로 확대되었다
- [ ] +/- 버튼 터치 타겟이 44×44dp 이상이다

#### 테스트 방법
1. 각 슬라이더의 +/- 버튼 탭 → 단계별 값 변경 확인
2. 값이 min/max에서 더 이상 증감하지 않는지 확인
3. 시각적으로 슬라이더가 더 두꺼워졌는지 확인

### 🔴 S2-C2: 터치 타겟 48dp 확보

- **우선순위**: P0 (WCAG 2.5.5)
- **대상 파일**: `lib/screens/song_list_screen.dart`, `lib/screens/prompter_screen.dart`
- **대상 위치**:

| 위젯 | 위치 | 현재 크기 | 목표 |
|---|---|---|---|
| `_CompactBtn` | `song_list_screen.dart:1855-1877` | 36×36 | 48×48 |
| `_BarIconButton` | `prompter_screen.dart:392-413` | 34×34 | 48×48 |
| `_PlayBtn`(LyricsScreen) | 삭제 대상(S1-A3) | - | - |
| 체크박스 | `song_list_screen.dart:1496-1506` | 20×20, shrinkWrap | 최소 48 탭 영역 |
| `_StepButton`(신규) | S2-C1 | 44×44 | 48×48 |

#### 작업 지시
1. `_CompactBtn`: `width: 36, height: 36` → `48, 48`. 아이콘 크기 20 → 24.
2. `_BarIconButton`: `width: 34, height: 34` → `48, 48`. 아이콘 18 → 22.
3. 체크박스: `MaterialTapTargetSize.shrinkWrap` 제거, `visualDensity: VisualDensity.standard`로 변경. 시각적 크기는 유지하되 탭 영역 확보.
4. `_SongTile`의 `_SmallActionButton`: `minHeight: 40` → `minHeight: 48`.

#### 구현 예시 (`_CompactBtn`)
```dart
class _CompactBtn extends StatelessWidget {
  final IconData icon;
  final VoidCallback onTap;
  final bool highlighted;
  final String? semanticsLabel; // S2-C3 연계

  const _CompactBtn({
    required this.icon,
    required this.onTap,
    this.highlighted = false,
    this.semanticsLabel,
  });

  @override
  Widget build(BuildContext context) {
    return SizedBox(
      width: 48,
      height: 48,
      child: GestureDetector(
        onTap: onTap,
        child: Container(
          decoration: BoxDecoration(
            color: highlighted ? AppColors.accent : AppColors.elevated,
            borderRadius: BorderRadius.circular(10),
            border: Border.all(
              color: highlighted ? AppColors.accent : AppColors.border,
            ),
          ),
          child: Icon(
            icon,
            size: 24,
            color: highlighted ? const Color(0xFF0A0A0A) : AppColors.textPrimary,
          ),
        ),
      ),
    );
  }
}
```

#### 완료 기준 (DoD)
- [ ] 모든 interactive 위젯의 터치 타겟이 48×48dp 이상이다
- [ ] 체크밝스 탭 영역이 48dp 이상이다 (shrinkWrap 제거)
- [ ] 레이아웃이 깨지지 않는다 (overflow 없음)

#### 테스트 방법
1. 각 버튼을 손가락으로 탭 → 정확히 탭되는지 확인
2. 작은 손가락/큰 손가락 모두 편하게 탭 가능한지 확인
3. 레이아웃에 overflow 경고 없는지 `flutter run` 로그 확인

### 🟠 S2-C3: Semantics / 스크린 리더 라벨

- **우선순위**: P1 (스크린 리더 사용자)
- **대상 파일**: 전체 위젯 파일
- **대상 위치**:
  - `GestureDetector` 기반 커스텀 버튼 전체 (`_CompactBtn`, `_BarIconButton`, `_SmallActionButton`, `_StepButton`)
  - `_SongTile`의 액션 버튼들
  - 아이콘 전용 버튼

#### 문제
`IconButton`은 `tooltip`이 있지만, `GestureDetector` 기반 커스텀 버튼은 **TalkBack/NVDA에 아무것도 전달하지 않음**.

#### 작업 지시
1. 모든 `GestureDetector` 기반 버튼을 `Semantics` 위젯으로 래핑.
2. `label`, `button: true`, `enabled: true` 속성 추가.
3. 상태가 있는 버튼(재생/일시정지 토글)은 `toggled`, `value` 속성 추가.
4. `ExcludeSemantics`로 중복 라벨 방지.

#### 구현 예시
```dart
// _CompactBtn
@override
Widget build(BuildContext context) {
  return Semantics(
    label: semanticsLabel ?? _defaultLabel(icon),
    button: true,
    enabled: true,
    child: SizedBox(
      width: 48,
      height: 48,
      child: GestureDetector(
        onTap: onTap,
        child: Container(
          // ... 기존 컨테이너
        ),
      ),
    ),
  );
}

String _defaultLabel(IconData icon) {
  if (icon == Icons.play_arrow) return '재생';
  if (icon == Icons.pause) return '일시정지';
  if (icon == Icons.stop) return '정지';
  if (icon == Icons.replay) return '처음부터';
  if (icon == Icons.fullscreen) return '전체화면';
  return '버튼';
}

// 토글 버튼 (재생/일시정지)
Semantics(
  label: _playing ? '일시정지' : '재생',
  button: true,
  toggled: _playing,
  child: _CompactBtn(icon: _playing ? Icons.pause : Icons.play_arrow, ...),
)
```

#### 완료 기준 (DoD)
- [ ] 모든 GestureDetector 기반 버튼에 Semantics label이 있다
- [ ] 토글 상태 버튼(재생/일시정지, 자동스크롤)에 `toggled` 속성이 있다
- [ ] NVDA/TalkBack에서 버튼 이름이 읽힌다 (Windows: 내레이터로 테스트)

#### 테스트 방법
1. Windows 내레이터(Win+Ctrl+Enter) 켜고 각 버튼 포커스 → 이름 읽히는지 확인
2. 슬라이더에 포커스 → "크기 슬라이더, 값 3" 같은 형태로 읽히는지 확인
3. `_SongTile`의 각 액션 버튼 이름 확인

### 🟠 S2-C4: 색 대비 검증 및 조정

- **우선순위**: P1 (WCAG AA 4.5:1)
- **대상 파일**: `lib/theme/app_theme.dart`, 전체 UI

#### 문제
`textMuted #9CA3AF` on `surface #111111` 등 대비 미검증. WCAG AA(일반 텍스트 4.5:1, 대형 텍스트 3:1) 충족 여부 확인 필요.

#### 작업 지시
1. 현재 색 조합의 대비 비율 계산 (아래 표 참고).
2. 미달 조합은 색 조정.
3. 검증 결과를 `app_theme.dart`에 주석으로 문서화.

#### 검증 대상 조합

| 전경 | 배경 | 추정 비율 | AA(4.5:1) |
|---|---|---|---|
| `textPrimary #F5F5F5` | `background #0A0A0A` | ~18:1 | ✅ |
| `textMuted #9CA3AF` | `background #0A0A0A` | ~7:1 | ✅ |
| `textMuted #9CA3AF` | `surface #111111` | ~6.5:1 | ✅ |
| `textMuted #9CA3AF` | `elevated #1A1A1A` | ~5.5:1 | ✅ (근접) |
| `accent #EAB308` | `background #0A0A0A` | ~12:1 | ✅ |
| `#0A0A0A`(버튼 텍스트) | `accent #EAB308` | ~12:1 | ✅ |
| `textPrimary #F5F5F5` | `border #262626` 버튼 | ~12:1 | ✅ |
| `textMuted #9CA3AF` | `border #262626`(_SmallActionButton 비활성) | ~4:1 | ⚠️ 근접 |

#### 조정 제안
- `_SmallActionButton` 비활성 배경(`border #262626`)의 텍스트를 `textPrimary`로 유지하되, 비활성 시 `textMuted` 색 사용 검토 → 대비 재계산 후 필요시 `textMuted`를 약간 밝게(`#B0B7C0` 등).
- 작은 폰트(11~12px)는 AA 기준 4.5:1 엄격 적용.

#### 구현 예시
```dart
class AppColors {
  static const background = Color(0xFF0A0A0A);
  static const surface = Color(0xFF111111);
  static const elevated = Color(0xFF1A1A1A);
  static const accent = Color(0xFFEAB308);
  static const accentHover = Color(0xFFFACC15);
  static const textPrimary = Color(0xFFF5F5F5);
  // 대비 강화: #9CA3AF → #B5BCC5 (surface 기준 7.5:1)
  static const textMuted = Color(0xFFB5BCC5);
  static const border = Color(0xFF262626);
  static const danger = Color(0xFFEF4444);
}
```

#### 완료 기준 (DoD)
- [ ] 모든 텍스트/배경 조합이 WCAG AA(4.5:1)를 충족한다
- [ ] `app_theme.dart`에 대비 비율 주석 추가
- [ ] 시각적 변경이 크지 않다 (무드 유지)

#### 테스트 방법
1. WebAIM Contrast Checker 또는 Flutter `ColorScheme` 디버그로 비율 확인
2. 실제 기기에서 저시력 시뮬레이션(Windows 설정: 고대비 모드)로 가독성 확인

### 🟢 S2-C5: 폰트 크기 단계 정교화 + 사용자 정의

- **우선순위**: P2 (사용자 맞춤)
- **대상 파일**: `lib/models/prompter_settings.dart`, `lib/screens/song_list_screen.dart`, `lib/screens/prompter_screen.dart`
- **연계**: S3-B2 (공통 로직 모듈화) 선행 권장, 단 Sprint 2에서 임시 적용 가능

#### 문제
폰트 크기 단계가 5단계 고정(18/24/32/42/56). 간격이 불균등(+6, +8, +10, +14). 저시력 프리셋은 더 촘촘하게, 무대용은 더 크게 등 사용자 맞춤이 어려움.

#### 작업 지시
1. `PrompterSettings`에 사용자 정의 폰트 크기(`double? customFontSizePt`) 필드 추가.
2. 폰트 크기 모델을 `fontSizeLevel`과 `customFontSizePt`를 모두 지원하도록 확장.
3. 슬라이더 단계를 더 촘촘하게 조정 (예: 7단계: 18/22/28/36/44/56/72).
4. 설정 패널에 "사용자 정의 크기" 옵션 추가 (직접 pt 입력 또는 세밀한 슬라이더).

#### 구현 예시
```dart
class PrompterSettings {
  final double fontSizeLevel;
  final double? customFontSizePt; // 신규
  // ... 기존 필드

  double get effectiveFontSizePt =>
      customFontSizePt ?? _defaultFontSizeForLevel(fontSizeLevel);

  static double _defaultFontSizeForLevel(double level) {
    if (level <= 1) return 18;
    if (level <= 2) return 22;
    if (level <= 3) return 28;
    if (level <= 4) return 36;
    if (level <= 5) return 44;
    if (level <= 6) return 56;
    return 72; // level 7
  }
}
```

#### 완료 기준 (DoD)
- [ ] 폰트 크기 단계가 7단계로 세분화되었다
- [ ] 사용자가 pt 값을 직접 지정할 수 있다 (설정 패널)
- [ ] 기존 v0.5 설정과 호환된다 (기본값 유지)

#### 테스트 방법
1. 각 단계(1~7)에서 폰트 크기가 예상대로 변경되는지 확인
2. 사용자 정의 pt(예: 40) 입력 → 해당 크기 적용 확인
3. 앱 재시작 후 사용자 정의 값 유지 확인

#### 리스크
- 기존 단계 변환 로직이 3중 복제 → S3-B2에서 통합 필요. 본 스프린트에서는 임시로 각 파일에 반영 후 Sprint 3에서 정리.
