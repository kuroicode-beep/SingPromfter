#### 구현 예시
```dart
// lib/theme/prompter_levels.dart

/// 프롬프터 폰트 크기/줄간격 단계 변환 공통 로직.
/// 모든 화면에서 이 모듈을 통해 일관된 변환을 사용한다.
class PrompterLevels {
  PrompterLevels._();

  static const double minLevel = 1;
  static const double maxLevel = 7;

  /// 폰트 크기 단계 → pt
  static double fontSizeForLevel(double level) {
    if (level <= 1) return 18;
    if (level <= 2) return 22;
    if (level <= 3) return 28;
    if (level <= 4) return 36;
    if (level <= 5) return 44;
    if (level <= 6) return 56;
    return 72;
  }

  /// 줄간격 단계 → 배수
  static double lineHeightForLevel(double level) {
    if (level <= 1) return 1.4;
    if (level <= 2) return 1.6;
    if (level <= 3) return 1.9;
    if (level <= 4) return 2.2;
    if (level <= 5) return 2.5;
    if (level <= 6) return 2.8;
    return 3.2;
  }

  /// pt → 폰트 크기 단계 (역변환)
  static double levelForFontSize(double pt) {
    if (pt <= 18) return 1;
    if (pt <= 22) return 2;
    if (pt <= 28) return 3;
    if (pt <= 36) return 4;
    if (pt <= 44) return 5;
    if (pt <= 56) return 6;
    return 7;
  }

  /// pt → 줄간격 단계 (역변환)
  static double levelForLineHeight(double value) {
    if (value <= 1.4) return 1;
    if (value <= 1.6) return 2;
    if (value <= 1.9) return 3;
    if (value <= 2.2) return 4;
    if (value <= 2.5) return 5;
    if (value <= 2.8) return 6;
    return 7;
  }

  /// 속도 단계 → 스크롤 px per tick
  static double scrollDeltaForSpeed(double speedLevel) => speedLevel * 1.4;
}
```

### 🟠 S3-B3: 저장소 분리 (SharedPreferences → JSON 파일)

- **우선순위**: P1
- **대상 파일**: `lib/repository/song_repository.dart`

#### 문제
`saveSongs`(`song_repository.dart:94`)가 `_songs` 전체를 매번 재직렬화. 가사는 이미 파일로도 저장 중 → **이중 저장**. SharedPreferences는 작은 값용(~수십 KB 권장). 곡이 수십 개만 돼도 한계.

#### 작업 지시

**1단계: 곡 메타 저장소 분리**
1. `lib/repository/song_meta_store.dart` 신규 작성.
2. 곡 메타(가사 텍스트 제외)를 단일 JSON 파일(`data/songs.json`)로 저장.
3. `lyricsText`는 메모리 캐시로만 보관, 디스크에는 `txt/` 파일이 단일 소스.

```dart
// lib/repository/song_meta_store.dart
class SongMetaStore {
  // songs.json에 가사 텍스트 없이 메타만 저장
  Future<List<Song>> load() async {
    final file = File('${(await _dataDir).path}/songs.json');
    if (!await file.exists()) return [];
    final raw = await file.readAsString();
    final list = jsonDecode(raw) as List<dynamic>;
    return list.map((e) => SongMeta.fromJson(e)).toList();
  }

  Future<void> save(List<Song> songs) async {
    final file = File('${(await _dataDir).path}/songs.json');
    final raw = jsonEncode(songs.map((s) => s.toMetaJson()).toList());
    await file.writeAsString(raw);
  }
}
```

**2단계: 마이그레이션 로직**
1. 기존 SharedPreferences 데이터를 새 JSON 파일로 일회성 마이그레이션.
2. 마이그레이션 후 SharedPreferences 키(`singpromfter_songs`) 삭제.
3. 기존 `loadSongs`의 마이그레이션 로직(`:54-83`) 제거.

**3단계: 설정/큐는 SharedPreferences 유지**
- 설정(`singpromfter_settings`), 큐(`singpromfter_queue`), 마지막 곡 ID(`singpromfter_last_song_id`)는 작으므로 SharedPreferences 유지 가능.
- 단, 구조 개선을 위해 별도 클래스(`SettingsStore`, `QueueStore`)로 캡슐화.

#### 데이터 흐름 (변경 후)

```
[곡 메타]  songs.json  ←→ SongMetaStore
[가사]     txt/*.txt   ←→ LyricsStore (파일 직접)
[반주]     mp3/*.mp3   (참조만, 복사본)
[설정]     SharedPreferences ←→ SettingsStore
[큐]       SharedPreferences ←→ QueueStore
```
