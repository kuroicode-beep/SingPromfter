#### 구현 예시 (방안 A)
```dart
Future<void> _deleteSong(Song song) async {
  // ... 기존 확인 다이얼로그 ...

  // 즉시 메모리/스토어에서 제거
  final backup = song; // 복구용 백업

  await _repo.softDeleteSong(song);
  // ... 상태 갱신 ...

  if (!mounted) return;
  ScaffoldMessenger.of(context).showSnackBar(
    SnackBar(
      content: Text('"${song.title}" 삭제됨'),
      duration: const Duration(seconds: 10),
      action: SnackBarAction(
        label: '실행 취소',
        onPressed: () async {
          await _repo.restoreSong(backup, backupFiles);
          // ... 상태 갱신 ...
        },

    ),
```

### 🟢 S4-D7: 일괄 등록

- **우선순위**: P2
- **대상 파일**: 신규 다이얼로그
- **버전**: v0.9.0

#### 문제
곡 하나당 다이얼로그 1회. 50곡 입력 시 비효율.

#### 작업 지시
1. "폴더 선택" 버튼으로 디렉토리 선택.
2. 폴더 내 txt + mp3 파일을 자동 매칭 (파일명 기준).
3. 매칭 결과를 표(체크박스)로 표시, 사용자가 선택적으로 등록.
4. 매칭 규칙: `제목.txt` + `제목_mr1.mp3` 등 기존 명명 규칙 활용.

#### 매칭 알고리즘 예시
```dart
List<BatchMatch> _matchFiles(List<File> files) {
  final txtFiles = files.where((f) => f.path.endsWith('.txt'));
  final mp3Files = files.where((f) => f.path.endsWith('.mp3'));
  final matches = <BatchMatch>[];

  for (final txt in txtFiles) {
    final stem = basenameWithoutExtension(txt.path);
    final tracks = <int, String>{};
    for (final slot in [1, 2, 3]) {
      final pattern = RegExp('^${RegExp.escape(stem)}_mr$slot\.mp3\$');
      final match = mp3Files.firstWhereOrNull((f) => pattern.hasMatch(basename(f.path)));
      if (match != null) tracks[slot] = match.path;
    }
    matches.add(BatchMatch(title: stem, lyricsPath: txt.path, trackPaths: tracks));
  }
  return matches;
}
```

#### 완료 기준 (DoD)
- [ ] 폴더 선택 후 txt/mp3가 자동 매칭된다
- [ ] 매칭 결과를 사용자가 확인/선택할 수 있다
- [ ] 선택된 곡이 일괄 등록된다
- [ ] 진행률이 표시된다

#### 테스트 방법
1. 폴더에 "노래1.txt", "노래1_mr1.mp3", "노래2.txt" 준비
2. 일괄 등록 실행 → 2개 곡 매칭 (노래1은 반주 포함)
3. 등록 완료 후 곡 목록 확인

#### 리스크
