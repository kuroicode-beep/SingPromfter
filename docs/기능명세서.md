# SingPromfter 기능 명세서

- 문서 버전: 1.0
- 작성일: 2026-03-16
- 기준 앱 버전: `1.0.1+1`
- 기준 소스: 현재 `lib/` 구현

## 1. 문서 목적

본 문서는 SingPromfter 앱의 현재 구현 상태를 기준으로 사용자 기능, 화면 동작, 데이터 저장 구조, 제약사항을 정리한 기능 명세서이다.  
기획안이 아니라 실제 코드에 반영된 동작을 기준으로 작성한다.

## 2. 제품 개요

SingPromfter는 저시력 사용자를 고려한 노래 프롬프터 앱이다. 사용자는 가사 `txt` 파일과 반주 `mp3` 파일을 등록하고, 곡 목록에서 원하는 곡을 선택해 가사 표시, 반주 재생, 예약 큐 관리, 전체화면 프롬프터 기능을 사용할 수 있다.

## 3. 지원 범위

### 3.1 핵심 지원 기능

- 곡 등록, 수정, 삭제
- 곡 검색, 즐겨찾기, 전체/즐겨찾기 필터
- 곡별 가사 표시
- 곡별 반주 0~3개 등록 및 슬롯 선택
- 반주 재생, 일시정지, 정지, 처음부터 재생, 위치 이동, 볼륨 조절, 재생 속도 조절
- 반주 라벨 사용자 정의 및 시작/끝 지점 설정
- 예약 큐 추가, 드래그 핸들 재정렬, 개별 삭제, 전체 비우기, 곡 종료 후 다음 예약곡 자동 재생
- 7단계 글자 크기, 사용자 정의 pt, 줄 간격, 자동 스크롤 속도, 글꼴, 굵게 설정
- 접근성 프리셋 적용
- 전체화면 프롬프터 진입 및 조작
- 설정/곡 목록/예약 큐/마지막 선택 곡 로컬 저장 및 재실행 복원
- zip 백업 내보내기/가져오기
- 폴더 기반 txt/mp3 일괄 등록

### 3.2 대상 플랫폼

- Flutter 기반 데스크톱 앱
- 로컬 파일 복사 기능 기준으로 Windows 데스크톱 환경을 주 사용 시나리오로 본다
- Web 전용 분기 코드는 제거했으며, 현재 검증 대상은 Windows 빌드이다

## 4. 사용자 화면 구성

### 4.1 메인 화면

- 앱 시작 시 기본 진입 화면이다.
- 폭이 넓으면 좌측 곡 목록, 우측 가사/재생 패널의 2단 레이아웃을 사용한다.
- 폭이 좁으면 상단 곡 목록, 하단 가사/재생 패널의 세로 분할 레이아웃을 사용한다.
- 우측 상단 플로팅 버튼으로 곡 등록을 시작한다.

### 4.2 곡 목록 패널

- 등록된 곡 목록을 카드 형태로 표시한다.
- 각 곡 카드에서 다음 액션을 제공한다.
- 목록 상단 검색바로 제목을 실시간 필터링하며, 한글 초성 검색도 지원한다.
- 전체/즐겨찾기 필터 칩으로 즐겨찾기 곡만 볼 수 있다.
- `선택`: 해당 곡을 현재 곡으로 로드
- `재생`: 해당 곡을 로드한 뒤 현재 선택된 반주 슬롯 또는 기본 슬롯으로 즉시 재생
- `예약`: 예약 큐에 추가
- `수정`: 곡 제목/가사/반주 수정
- `삭제`: 곡 및 관련 파일 삭제
- `별표`: 즐겨찾기 추가/해제
- 선택된 곡은 강조 표시된다.
- 선택된 곡에 반주가 있으면 `반주1~반주3` 슬롯 버튼이 노출된다.

### 4.3 가사/재생 패널

- 선택된 곡의 가사를 중앙 정렬로 표시한다.
- 가사가 없으면 안내 문구를 표시한다.
- 반주 제어 바와 접근성 설정 바를 하단에 제공한다.
- 주요 조작 버튼과 슬라이더는 48dp 터치 타겟과 스크린 리더 라벨을 제공한다.
- 예약 큐가 비어 있지 않으면 하단에 예약 큐 패널을 노출한다.

### 4.4 전체화면 프롬프터

- 현재 곡 가사를 검정 배경 전체화면으로 표시한다.
- 상단에는 닫기 버튼과 곡 제목을 표시한다.
- 하단에는 자동 스크롤 토글, +/- 보조 버튼이 있는 글자 크기/줄 간격/속도 조절, 수동 위/아래 이동 버튼을 제공한다.
- 화면 탭으로 컨트롤 표시/숨김을 전환한다.

## 5. 기능 상세

### 5.1 앱 초기화

- 저장된 곡 목록, 예약 큐, 프롬프터 설정, 마지막 선택 곡 ID를 불러온다.
- 마지막 선택 곡이 존재하면 해당 곡을 우선 선택한다.
- 마지막 선택 곡이 없고 곡 목록이 있으면 첫 번째 곡을 선택한다.
- 선택 곡이 있으면 저장된 반주 슬롯 우선순위에 따라 반주 준비를 시도한다.

### 5.2 곡 등록

#### 입력

- 가사 파일: `txt` 1개, 필수
- 반주 파일: `mp3` 계열 오디오 파일 0~3개, 선택
- 곡 제목: 기본값은 가사 파일명에서 `.txt` 제거한 값

#### 처리 규칙

- `txt` 외 확장자는 거부한다.
- 가사 파일은 우선 UTF-8로 디코딩하고 실패 시 `latin1`로 대체한다.
- 동일 제목 곡은 등록할 수 없다.
- 곡 ID는 UUID로 생성한다.
- 가사는 앱 문서 디렉터리 하위 `data/txt/`에 저장한다.
- 반주는 슬롯별로 앱 문서 디렉터리 하위 `data/mp3/`에 복사한다.
- 반주 파일명은 `{제목}_mr{slot}.mp3` 규칙을 사용한다.
- 파일명에 사용할 수 없는 문자는 공백으로 치환하고 후행 점/공백은 제거한다.

#### 결과

- 곡 목록에 새 곡을 추가한다.
- 새 곡을 현재 선택 곡으로 로드한다.
- 등록 성공 메시지를 표시한다.

#### 예외/제약

- Windows 데스크톱 검증을 기준으로 하며, Web 전용 파일 처리 분기는 지원 범위에서 제외한다.
- 가사 파일 내용을 읽지 못하면 등록을 중단한다.

### 5.3 곡 수정

#### 수정 가능 항목

- 곡 제목
- 가사 텍스트
- 반주 슬롯별 파일 교체

#### 처리 규칙

- 동일 제목의 다른 곡이 있으면 수정할 수 없다.
- 제목이 바뀌면 가사 파일명을 새 제목 기준으로 다시 생성한다.
- 제목이 바뀌고 기존 반주 파일이 존재하면 새 제목 기준 파일명으로 복사 후 이전 파일을 삭제한다.
- 특정 슬롯에 새 반주를 지정하면 해당 슬롯 파일을 교체한다.
- 수정한 곡이 현재 선택 곡이면 수정 완료 후 다시 로드한다.

#### 결과

- 곡 정보와 수정 시각을 갱신한다.
- 성공 시 안내 메시지를 표시한다.

### 5.4 곡 삭제

#### 처리 규칙

- 곡 정보는 즉시 목록에서 제거하고, 10초 실행 취소 시간이 지난 뒤 가사 파일 및 연결된 반주 파일을 영구 삭제한다.
- 현재 저장 구조와 레거시 저장 구조를 모두 삭제 대상으로 확인한다.
- 예약 큐에 포함된 해당 곡 항목도 함께 제거한다.
- 삭제된 곡이 현재 선택 곡이면 다음 곡 또는 선택 없음 상태로 전환한다.
- 남은 곡이 없으면 마지막 선택 곡 ID를 제거한다.

### 5.5 곡 선택 및 반주 슬롯 선택

- 곡 선택 시 현재 곡, 현재 반주 슬롯, 재생 위치를 초기화한다.
- 반주 슬롯 결정 우선순위는 다음과 같다.
- 명시적으로 요청된 슬롯
- 곡별 마지막 선택 슬롯
- 전역 마지막 선택 슬롯
- 현재 곡의 첫 번째 사용 가능 슬롯
- 반주 슬롯이 없으면 가사 전용 곡으로 동작한다.
- 사용자가 슬롯 버튼을 누르면 해당 곡의 마지막 선택 슬롯을 저장한다.

### 5.6 반주 재생 제어

#### 제공 기능

- 재생/일시정지 토글
- 정지 후 0초 이동
- 처음부터 다시 재생
- 다음 예약곡 수동 이동
- 진행 위치 슬라이더 탐색
- +/- 보조 버튼이 있는 볼륨 조절

#### 처리 규칙

- 반주 파일이 정상 준비된 경우에만 실제 재생한다.
- 반주가 없으면 가사만 표시되며 재생 버튼 클릭 시 안내 메시지를 표시한다.
- 반주 파일 경로를 찾지 못하면 오류 메시지를 표시한다.
- 재생 상태, 현재 위치, 전체 길이는 오디오 플레이어 스트림을 통해 갱신한다.

### 5.7 자동 스크롤

- 메인 화면 가사 패널과 전체화면 프롬프터 모두 자동 스크롤을 지원한다.
- 속도 값이 `0`보다 크고 스크롤 가능한 상태일 때만 동작한다.
- 공통 자동 스크롤 주기와 속도 계수를 기준으로 `speedLevel`에 비례해 아래로 이동한다.
- 메인 화면에서는 반주 재생 중이거나 가사 전용 곡에서 속도가 0보다 클 때 자동 스크롤을 시작한다.
- 전체화면 프롬프터에서는 사용자가 자동 스크롤 토글 버튼으로 켜고 끌 수 있다.

### 5.8 예약 큐

#### 제공 기능

- 선택한 곡 예약 추가
- 예약 항목 개별 삭제
- 예약 큐 전체 비우기
- 드래그 핸들 기반 예약 순서 재정렬
- 현재 곡 종료 후 다음 예약곡 자동 재생

#### 처리 규칙

- 예약 시 곡별 마지막 선택 슬롯이 유효하면 해당 슬롯을 함께 저장한다.
- 유효한 슬롯이 없고 반주가 있으면 첫 번째 슬롯을 저장한다.
- 반주가 없으면 슬롯 없이 예약할 수 있다.
- 예약 순서는 추가 순서를 유지한다.
- 현재 곡 재생 완료 이벤트가 발생하면 큐의 첫 항목을 꺼내 자동 재생을 시도한다.
- 큐에 있는 곡이 삭제되어 더 이상 존재하지 않으면 해당 항목은 건너뛴다.

#### 현재 UI 제약

- 예약 큐는 접근성 확보를 위해 세로 리스트로 표시된다.
- 예약 순서 변경은 각 항목의 드래그 핸들로 수행한다.

### 5.9 접근성 설정

#### 조정 항목

- 글자 크기 레벨: 1~7
- 사용자 정의 글자 크기(pt): 12~120
- 줄 간격 레벨: 1~7
- 자동 스크롤 속도: 0~10
- 글꼴 선택
- 굵게 표시 여부
- 슬라이더 양옆 +/- 버튼 보조 조작
- 볼륨: 0~1

#### 매핑 규칙

- 글자 크기: `1=18pt`, `2=22pt`, `3=28pt`, `4=36pt`, `5=44pt`, `6=56pt`, `7=72pt`
- 사용자 정의 글자 크기가 있으면 단계값보다 우선한다.
- 줄 간격: `1=1.35`, `2=1.5`, `3=1.7`, `4=1.95`, `5=2.2`, `6=2.45`, `7=2.75`

#### 글꼴 옵션

- `System Default`
- `Malgun Gothic`
- `Segoe UI`
- `Monospace`

#### 굵게 옵션

- 활성화 시 `FontWeight.w800`
- 비활성화 시 `FontWeight.w500`

### 5.10 접근성 프리셋

- `표준`: 글자 3, 줄간격 3, 속도 2, 기본 글꼴, 굵게 끔
- `저시력`: 글자 5, 줄간격 5, 속도 3, `Malgun Gothic`, 굵게 켬
- `원거리`: 글자 7, 줄간격 6, 속도 2, `Malgun Gothic`, 굵게 켬

### 5.11 전체화면 프롬프터

#### 진입 방식

- 메인 화면 하단 전체화면 버튼
- 키보드 `F5`

#### 동작

- `SystemUiMode.immersiveSticky` 적용
- 현재 곡 가사를 중앙 정렬로 표시
- 메인 화면의 글자 크기/줄 간격/속도/글꼴/굵게 설정을 전달
- 프롬프터 안에서 변경한 글자 크기/줄 간격/속도는 메인 설정에도 반영
- 자동 스크롤 버튼으로 프롬프터 내부 스크롤 자동 이동 제어

#### 종료 방식

- 닫기 버튼
- 키보드 `Escape`

#### 종료 처리

- `SystemUiMode.edgeToEdge` 복원

### 5.12 단축키

- `Space`: 재생/일시정지
- `F5`: 전체화면 프롬프터 열기
- `Escape`: 전체화면 프롬프터 닫기

## 6. 데이터 저장 명세

### 6.1 SharedPreferences 키

- `singpromfter_songs`: 곡 목록 JSON
- `singpromfter_settings`: 프롬프터 설정 JSON
- `singpromfter_queue`: 예약 큐 JSON
- `singpromfter_last_song_id`: 마지막 선택 곡 ID

### 6.2 곡 데이터 구조

`Song`

- `id`: 문자열 UUID
- `title`: 곡 제목
- `lyricsPath`: 가사 파일 경로 또는 파일명
- `lyricsText`: 가사 본문
- `backingTracks`: 반주 목록
- `createdAt`: 생성 시각
- `updatedAt`: 수정 시각

`BackingTrack`

- `slot`: 반주 슬롯 번호 `1~3`
- `fileName`: 저장된 반주 파일명
- `label`: 표시용 라벨

`QueueItem`

- `songId`: 예약된 곡 ID
- `selectedTrackSlot`: 예약 시 함께 저장된 반주 슬롯
- `queuedAt`: 예약 시각

`PrompterSettings`

- `fontSizeLevel`
- `lineHeightLevel`
- `speedLevel`
- `volume`
- `lastSelectedTrackSlot`
- `fontFamily`
- `boldText`
- `lastSelectedTrackSlotBySong`

### 6.3 파일 저장 경로

현재 저장 구조

- 가사: `{Documents}/data/txt/{정제된제목}.txt`
- 반주: `{Documents}/data/mp3/{정제된제목}_mr{slot}.mp3`

레거시 호환 경로

- 가사: `{Documents}/lyrics/{id}.txt`
- 반주: `{Documents}/mr/{fileName}`

## 7. 마이그레이션 및 호환성

### 7.1 가사 텍스트 보완

- 저장된 곡에 `lyricsText`가 비어 있으면 다음 순서로 가사를 찾는다.
- `lyricsPath`
- 현재 가사 디렉터리의 `{id}.txt`
- 현재 가사 디렉터리의 `{title}.txt`
- 레거시 가사 디렉터리의 `{id}.txt`
- 가사를 찾으면 `lyricsText`를 메모리에 보완하고 SharedPreferences에 다시 저장한다.

### 7.2 반주 파일 경로 보완

- 반주 재생 시 현재 저장 경로에서 우선 검색한다.
- 없으면 레거시 반주 디렉터리에서 재검색한다.

### 7.3 레거시 MR 필드 호환

- 기존 데이터에 `mrFileName`만 있고 `backingTracks`가 없으면 슬롯 1의 반주로 변환한다.

## 8. 예외 처리 및 메시지 정책

- 지원하지 않는 파일 형식 선택 시 즉시 사용자 메시지를 표시한다.
- 동일 제목 중복 시 저장을 차단하고 제목 변경을 안내한다.
- 반주 파일을 찾을 수 없거나 재생할 수 없으면 안내 메시지를 표시한다.
- Windows 파일 선택/복사 실패 시 제한사항을 안내한다.
- 모든 주요 성공/실패 이벤트는 `SnackBar`로 피드백한다.

## 9. 현재 구현 기준 비포함 항목

- 사용자 계정

## 10. 품질 보증

- `flutter analyze`는 강화된 `analysis_options.yaml` 기준으로 실행한다.
