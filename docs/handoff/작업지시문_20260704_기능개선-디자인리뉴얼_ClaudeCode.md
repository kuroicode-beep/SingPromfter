## 1. 배경 및 목표

v1.0.1은 기능·품질(테스트 20개, CI) 면에서 안정화되었으나, UI는 단일 화면(좌측 곡 목록 + 우측 프롬프터) 구조에 노란색 강조의 기본 Material 다크 테마다.

`asset/stitch_ui.zip`의 **Core Precision Dark** 시안을 기준으로 다음을 달성한다.

1. **디자인 시스템 전환**: 토널 레이어링 기반 딥 다크 + 바이올렛/블루 강조 팔레트로 교체
2. **레이아웃 리뉴얼**: 좌측 네비게이션 레일 + 3열 작업 화면(곡 목록 / 프롬프터 / 예약 큐)
3. **기능 개선**: 가수 필드, 프롬프터 현재 줄 하이라이트, 예약/시작 명시 버튼, 재생 진행률 표시 등 시안에서 도출된 실용 기능

### 시안 요약

| 시안 | 내용 |
|---|---|
| `desktop_1` | 홈/재생 화면: 좌 네비 레일, 상단 Now Playing 바 + [곡 시작], 중앙 3줄 프롬프터(현재 줄 글로우), 하단 진행률 바, 우측 예약 큐(순번 + NOW 배지) |
| `desktop_2` | 곡 검색 화면: 대형 검색창, 필터 칩, 결과 리스트(번호/제목/가수/[예약]/[시작] 버튼) |
| `core_precision_dark/DESIGN.md` | 색상 토큰, 타이포, 4px 스페이싱 리듬, 라운딩, 컴포넌트 규칙 |

## 2. 최우선 원칙 — 접근성 오버라이드 (필독)

시안은 "파워 유저용 고밀도 UI"를 전제로 하여 **본 앱의 저시력 사용자 기준과 충돌하는 부분이 있다.**
시안의 **시각 언어(색, 토널 레이어링, 글로우, 라운딩)는 채택**하되, 아래 항목은 반드시 앱 기준으로 오버라이드한다.

| 항목 | 시안 값 | 적용 값 (강제) |
|---|---|---|
| 최소 폰트 크기 | label 10~12px | **모든 텍스트 16px 이상** (보조 라벨 포함) |
| 터치/클릭 타겟 | 24px 칩 등 | **50dp 이상** (기존 48dp에서 상향) |
| 상태 구분 | 색상 위주 | **색상 + 텍스트 라벨 병행** (예: NOW 배지, 재생중 텍스트) |
| 대비 | AAA 지향 | 신규 팔레트 전 조합 **WCAG AA(4.5:1) 이상 검증** 후 `app_theme.dart` 주석에 대비값 기록 (기존 방식 유지) |
| JetBrains Mono 라벨 | 10px 모노 캡션 | 모노 폰트 사용은 선택, 크기는 16px 이상 |

기존에 확보한 접근성 기능(7단계 글자 크기, 프리셋 3종, +/- 보조 버튼, Semantics 라벨, 키보드 단축키)은 **회귀 없이 유지**한다.

## 3. Sprint 6 — 디자인 시스템 전환

### S6-1. 컬러 토큰 교체 (P0)

- 대상: `lib/theme/app_theme.dart`
- `AppColors`를 시안 토큰 기반으로 재정의한다.

```dart
// Core Precision Dark 기반 (DESIGN.md 참조)
background:        #0E0D15   // surface-container-lowest (전역 배경)
surface:           #13131A   // 기본 표면
surfaceContainer:  #1F1F27   // 카드/패널
elevated:          #2A2931   // 팝오버/모달/호버
primary:           #C2C1FF   // 텍스트·아이콘 강조 (바이올렛)
primaryContainer:  #5856D6   // 주요 버튼 배경 (흰 텍스트)
secondary:         #ADC6FF   // 보조 강조 (블루)
tertiary:          #FFB785   // 예약/대기 상태 강조 (오렌지)
onSurface:         #E4E1EC
onSurfaceVariant:  #C7C4D6   // 기존 textMuted 대체
outline:           #464554   // 기존 border 대체
danger:            #FFB4AB   // error 토큰
```

- 기존 노란색(`#EAB308`) 참조를 전수 검색하여 교체한다. `accent` 심볼을 유지하고 값만 바꾸는 방식 허용(변경 범위 최소화).
- 프롬프터 화면(순수 검정 배경 + 고대비 가사)은 **기존 색을 유지**한다. 무대용 최대 대비가 우선이며, 강조색만 primary로 통일.

**DoD**: `flutter analyze` 통과, 노란색 하드코딩 잔존 0건, 신규 대비값 주석 기록.

### S6-2. 셰이프·표면 규칙 적용 (P1)

- 카드/패널: 배경 `surfaceContainer`, 라운딩 16px, 테두리 1px `outline`(또는 white 10% opacity)
- 버튼/입력 필드: 라운딩 8px
- 리스트: 교차 배경 대신 1px 구분선. **활성 행 = primaryContainer 10% 배경 + 좌측 3px 세로 인디케이터 + 텍스트 상태 라벨**
- 호버: 표면 밝기 +5% 또는 좌측 2px 강조 보더

**DoD**: `song_tile.dart`, `queue_panel.dart`, `song_list_panel.dart`, `prompter_bottom_bar.dart`에 일관 적용.

### S6-3. 타이포그래피 (P1)

- Pretendard 폰트 도입을 검토한다(OFL 라이선스, 무료). 도입 시 `assets/fonts`에 추가하고 기본 폰트로 지정, 기존 맑은 고딕/Segoe UI는 프롬프터 글꼴 선택 옵션으로 유지.
- 계층: 화면 제목 24px/600, 리스트 제목 18px/600, 본문 16px/400, 보조 정보 16px/400 + onSurfaceVariant 색.
- 프롬프터 글꼴 선택·굵게 기능은 그대로 유지.

**DoD**: 16px 미만 텍스트 0건 (위젯 테스트로 스팟 체크).

## 4. Sprint 7 — 레이아웃 리뉴얼

### S7-1. 네비게이션 레일 도입 (P0)

- 대상: `lib/widgets/song_list_screen_view.dart` (구조 변경의 중심)
- AppBar 우측 아이콘 3개(일괄 등록/백업 내보내기/가져오기) + FAB(곡 등록) 구조를 폐기하고, 좌측 고정 네비게이션 레일로 재배치한다.

```
[레일 240px (좁은 창 72px 아이콘 모드)]
  SingPromfter (로고/타이틀)
  ─ 홈 (곡 목록 + 프롬프터)
  ─ 곡 검색
  ─ 즐겨찾기
  ─ 설정 (백업 내보내기/가져오기, 일괄 등록, 프롬프터 기본값)
  [곡 등록] 주요 버튼 (primaryContainer, 레일 하단)
```

- 시안의 "Core Precision / Vocalist 01" 등 브랜딩 텍스트는 참고용이므로 사용하지 않는다. 앱명 SingPromfter 유지.
- 레일 항목은 아이콘 + 텍스트 병행, 활성 항목은 배경 틴트 + 좌측 인디케이터.

**DoD**: 기존 AppBar 액션·FAB 기능 전부 레일/설정 화면에서 접근 가능, 키보드 단축키 회귀 없음.

### S7-2. 홈 화면 3열 구성 (P0)

- desktop_1 시안 기준. 와이드(≥ 기존 `wideLayoutBreakpoint`) 레이아웃:

```
[레일] | [곡 목록 360px] | [프롬프터 미리보기 (확장)] | [예약 큐 320px]
```

- 상단 상태 바: `Now Playing: {곡 제목}` + 선택 반주 슬롯 라벨 + [곡 시작](전체화면 프롬프터 진입) 버튼. 재생 중이 아니면 `선택된 곡: {제목}` 표기.
- 좁은 창: 기존 상하 분할 유지 + 큐는 탭/시트로 접근(현행 동작 보존 우선, 과도한 재작성 금지).
- 시안의 Sound Settings 카드 위치에 기존 볼륨/재생 속도 컨트롤(`prompter_bottom_bar.dart`)을 카드 스타일로 정리.

**DoD**: 와이드/좁은 창 양쪽에서 기존 시나리오(선택→반주 선택→재생→큐 자동 진행) 정상 동작.

### S7-3. 곡 검색 화면 (P1)

- desktop_2 시안 기준. 레일의 "곡 검색" 진입 시:
  - 상단 대형 검색창(포커스 시 secondary 보더 + 글로우), 라벨은 필드 위 상시 표시
  - 필터 칩(높이 50dp): **전체 / 즐겨찾기 / 반주 있음 / 최근 등록** — 시안의 장르 칩(트로트/POP)은 데이터가 없으므로 제외
  - 결과 행: 제목 / 가수(S8-1 완료 후) / 반주 슬롯 수 + **[예약] [시작]** 버튼(S8-3)
- 기존 `song_list_panel.dart`의 검색·필터 로직을 재사용한다. 별도 상태 중복 금지.

**DoD**: 검색/필터 결과가 홈 곡 목록과 동일 데이터 소스 공유, 위젯 테스트 1건 이상.

## 5. Sprint 8 — 기능 개선

### S8-1. 가수(artist) 필드 추가 (P1)

- 대상: `lib/models/song.dart`, 등록/수정 다이얼로그, `song_tile.dart`, 백업
- `Song`에 `artist` 필드(옵션, 기본 빈 문자열) 추가. `fromJson`은 필드 부재 시 빈 값 허용(기존 데이터 마이그레이션 불필요).
- 검색은 제목 + 가수 동시 매칭으로 확장.
- `BackupService` 내보내기/가져오기에 필드 포함, **구버전 백업 zip 가져오기 호환 유지**.

**DoD**: 구버전 JSON/백업 로드 테스트 통과, 검색 확장 단위 테스트 추가.

### S8-2. 프롬프터 현재 줄 하이라이트 모드 (P0 — 이번 리뉴얼의 핵심 가치)

- 대상: `lib/screens/prompter_screen.dart`, `prompter_auto_scroll_service.dart`
- desktop_1 시안의 3줄 컨텍스트 표시를 옵션 모드로 구현한다.
  - **하이라이트 모드**: 현재 줄 = 최대 크기 + primary 글로우(`shadows`로 표현), 이전/다음 줄 = 축소 + onSurfaceVariant 흐림. 자동 스크롤 진행에 따라 현재 줄 인덱스 이동.
  - **전체 모드(기존)**: 현행 전체 텍스트 스크롤 유지.
- 설정(프리셋 포함)에 모드 저장. 저시력 추천 프리셋의 기본값은 사용자 확인 전까지 기존 모드 유지.
- 현재 줄 판정은 자동 스크롤 타이머 기반(오디오 싱크 아님)임을 코드 주석과 설정 UI 문구에 명시 — 가사 타임스탬프 기능은 이번 범위 밖.

**DoD**: 두 모드 전환 즉시 반영, 가사 전용 곡(반주 없음)에서도 동작, 위젯 테스트 추가.

### S8-3. 리스트 행 [예약]/[시작] 명시 버튼 (P1)

- 대상: `lib/widgets/song_tile.dart`, `song_action_coordinator.dart`
- 현행 탭/컨텍스트 동작에 더해, 행 우측에 **[예약]**(큐 추가, tertiary 스타일)과 **[시작]**(즉시 재생+프롬프터, primaryContainer 스타일) 버튼을 상시 노출한다. 높이 50dp, 텍스트 라벨 필수.
- 기존 큐 로직(`song_queue_service.dart`) 재사용. 중복 예약 정책은 현행 규칙 유지.

**DoD**: 버튼 경유 예약/시작이 기존 경로와 동일 결과, Semantics 라벨 부여.

### S8-4. 예약 큐 시각 개선 (P2)

- 대상: `lib/widgets/queue_panel.dart`
- 순번(01, 02…) 표시, 재생 중 항목에 **NOW 배지(텍스트) + primaryContainer 틴트 + 좌측 인디케이터**.
- 드래그 재정렬·삭제·자동 진행 등 기존 동작 회귀 금지.
- "검색 결과 전체 예약" 버튼(시안의 '전체 곡 예약하기')은 필터된 결과를 일괄 큐 추가 — 확인 다이얼로그 필수.

**DoD**: 큐 관련 기존 테스트 통과 + 일괄 예약 로직 단위 테스트.

### S8-5. 재생 진행률 바 (P2)

- 대상: `lib/widgets/prompter_panel.dart`, 전체화면 프롬프터
- 프롬프터 하단에 현재 위치/전체 길이 진행률 바(primary 색, 두께 6px 이상) + 시간 텍스트(`m:ss / m:ss`, 16px 이상). 기존 시크 기능과 연결.

**DoD**: 재생/일시정지/시크 시 실시간 갱신.

### 범위 제외 (구현 금지 — 혼동 방지)
